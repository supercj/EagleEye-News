# EagleEye News 隐私说明

**最后更新：2025 年 6 月**

EagleEye News（以下简称「本扩展」）是一款本地运行的 Chrome/Edge 资讯聚合扩展。本扩展不提供账号系统，不采集、上传或出售任何个人信息。本隐私说明旨在向用户透明地介绍扩展的数据处理方式。

---

## 数据存储

本扩展仅使用浏览器内置的 `chrome.storage.local` API 在本地保存数据，包括：

- 用户启用的资讯来源和刷新间隔设置
- 用户导入的 RSS、Atom、JSON Feed 或 OPML 订阅源地址
- 收藏的文章 ID
- 已读的文章 ID
- 最近一次刷新时间和来源刷新状态
- 聚合到的文章标题、链接、来源、发布时间和短摘要

**以上数据仅保存在用户自己的浏览器中，不会传输到任何外部服务器。**

卸载扩展或清除浏览器扩展数据后，本地数据将被永久删除，无法恢复。

---

## 网络请求

本扩展仅在以下场景发起网络请求：

1. **获取资讯内容**：向用户启用的公开资讯来源发起请求，用于获取 RSS、Atom、JSON Feed、公开 API 或榜单页面内容。
2. **导入自定义订阅**：用户主动导入订阅源时，扩展会向对应源地址请求内容，以识别订阅名称和类型。

本扩展不会绕过登录、付费墙、反爬机制或访问限制，也不会复制来源网站的全文内容。

---

## 权限用途

| 权限 | 用途 |
| --- | --- |
| `storage` | 保存设置、订阅源、收藏、已读状态和文章列表 |
| `alarms` | 按用户设置的间隔在后台定时刷新资讯 |
| `permissions` | 导入自定义订阅源时，请求访问对应站点的权限 |
| 主机权限 | 访问内置资讯源和用户明确导入的订阅源 |

本扩展仅申请运行所必需的最小权限集，不会申请超出功能需要的权限。

---

## 第三方服务

本扩展当前不接入任何第三方分析、广告、追踪或 AI 服务。

若未来新增 AI 摘要等能力，将优先让用户自行提供 API Key，或在新增服务端方案前更新本隐私说明。

---

## 儿童隐私

本扩展不针对 13 岁以下儿童，也不会有意采集儿童的个人信息。
